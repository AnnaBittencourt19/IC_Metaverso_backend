##### 1. Importações Adicionais
O script agora importa classes locais:
- `EventoOnline`: Importada do arquivo `evento_online.py`
- `Evento`: Importada do arquivo `evento.py`
##### 2. Criação de Dados (Contexto Global)
- Antes de iniciar o servidor, são criados exemplos dessas classes (ex: `ev_online`, `ev2_online`, `ev`)
- Esses exemplos são armazenados em uma lista global (`eventos = [...]`) para que o _handler_ possa acessá-los durante as requisições
##### 3. Roteamento no `do_GET`
Dentro do `SimpleHandler`, o método `do_GET` agora implementa "rotas":
- A variável `self.path` contém o caminho da URL que o navegador acessou
- Uma estrutura `if/elif` é usada para verificar o valor de `self.path`:
    - `if self.path == "/"`: Define o conteúdo para a rota raiz
    - `elif self.path == "/eventos"`: Define o conteúdo para a rota `/eventos`
- Isso permite que o servidor envie respostas (páginas HTML) diferentes para URLs diferentes
- A resposta depende do caminho
##### 4. Geração de HTML Dinâmico (Rota `/eventos`)
Para a rota `/eventos` o HTML é construído:
1. **Inicialização:** Uma variável string vazia é criada (`eventos_html = ""`)
2. **Loop:** O código percorre a lista global `eventos` (`for ev in eventos:`)
3. **Adição:** A cada passagem do loop, uma `f-string` contendo o HTML de uma linha de tabela (`<tr>...</tr>`) com os dados do evento(`ev.id`, `ev.nome`, `ev.local`) é adicionada à variável `eventos_html`
4. **Inclusão:** No final, a variável eventos_html (agora contendo todas as linhas) é colocada dentro da estrutura principal `<table>`do HTML que será enviado
##### 5. Observações Importantes (dos comentários)
- **Acoplamento:** Esta forma de gerar HTML no backend (Python) mistura responsabilidades. O backend "sabe" sobre `<table>`, `<tr>`, etc. Em um cenário comum, o backend entregaria apenas os dados (em JSON) e o frontend (JavaScript) cuidaria de criar a tabela
- **end_headers():** É obrigatório. Se não for chamado, o navegador fica esperando por mais cabeçalhos e a página não carrega
- **f-strings:** Permitem inserir variáveis (ex: {ev.nome}) diretamente dentro de strings de forma simples, usando chaves {}
##### 6. Código com comentários

``` python
from http.server import HTTPServer, BaseHTTPRequestHandler
from evento_online import EventoOnline ## Importando a classe EventoOnline do arquivo evento_online.py
from evento import Evento ## Importando a classe Evento do arquivo evento.py

#Criação de alguns eventos para exibir no servidor
ev_online = EventoOnline("Live de Python")
ev2_online = EventoOnline("Live de JavaScript")
ev = Evento("Aula de Python", "Rio de Janeiro")
eventos = [ev_online, ev2_online, ev] # Lista para armazenar os eventos criados

class SimpleHandler(BaseHTTPRequestHandler):
    def do_GET(self):
        if self.path == "/": #if para rota raiz quando estivermos em localhost:8000/ ira aparecer o conteudo dentro do if  
            print ("Recebida uma requisição GET")
            self.send_response(200)
            self.send_header("Content-type", "text/html; charset=utf-8") 
            self.end_headers() 
            data = f"""
            <html>
                <head>
                    <title>Servidor HTTP Simples</title>
                </head>
                <body>
                    <h1>Olá, mundo!</h1>
                    <p> Diretorio: {self.path} </p>
                </body>
            </html>
            """.encode()
            self.wfile.write(data) 

        #if self.path vai fazer aquilo acontecer de acordo com o diretorio (/...) que estivermos acessando no navegador
        elif self.path == "/eventos": #if para rota status quando estivermos em localhost:8000/eventos ira aparecer o conteudo dentro do if
            self.send_response(200)
            self.send_header("Content-type", "text/html; charset=utf-8")
            self.end_headers() #Se não colocar o end_headers o navegador fica esperando mais informações, então sempre que colocar header tem que colocar o end_headers
            
            eventos_html = "" # inicializa a variavel eventos_html como string vazia

            for ev in eventos: #olha elementos dentro da lista eventos
                # para cada evento criado, cria uma linha na tabela com as informacoes do evento
                eventos_html += f"""
                        <tr>
                            <td> {ev.id} </td>
                            <td> {ev.nome} </td>
                            <td> {ev.local} </td>
                        </tr>
                """
            #mostrar informacoes dos eventos criados
            # insere o conteudo da variavel eventos_html dentro do html
            # em tabelas (table, tr, th)
            data = f""" 
            <html>
                <table>
                    <tr>
                        <th> ID </th>
                        <th> Nome </th>
                        <th> Local </th>
                    </tr>
            {eventos_html} 
            </html>
            """.encode()
            self.wfile.write(data)

server = HTTPServer(('localhost', 8000), SimpleHandler)
server.serve_forever() 

# f strings permite inserir variaveis dentro de strings de forma simples usando chaves {} 

```
